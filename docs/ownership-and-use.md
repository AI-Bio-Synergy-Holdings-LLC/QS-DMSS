# Ownership And Use

This page is intended to remove ambiguity for researchers, reviewers,
contributors, and sponsors using QS-DMSS.

## Project Stewardship

QS-DMSS is created, managed, and maintained by AI-Bio Synergy Holdings LLC.

The canonical public front door is:

```text
https://qs-dmss.studio
```

The public repository is:

```text
https://github.com/AI-Bio-Synergy-Holdings-LLC/QS-DMSS
```

The public Python package is:

```text
https://pypi.org/project/qs-dmss/
```

## Citation And Attribution

For research references, cite the Zenodo record that matches the work being
used:

- Project concept DOI: `10.5281/zenodo.20074924`
- Latest archived release DOI:
  `10.5281/zenodo.21270512` (`v0.10.1`)

Repository citation metadata lives in [`CITATION.cff`](../CITATION.cff), and
additional citation notes live in [`zenodo-citation.md`](zenodo-citation.md).

## Published Version Accreditation

For the published QS-DMSS package and archived releases, credit AI-Bio Synergy
Holdings LLC as the project creator, maintainer, and software author unless a
specific future release names additional authors or contributors.

Recommended short attribution:

```text
QS-DMSS, created and maintained by AI-Bio Synergy Holdings LLC.
```

Recommended reproducibility attribution:

```text
This work used QS-DMSS v0.10.1 / qs-dmss==0.10.1. The project concept DOI is
10.5281/zenodo.20074924; the v0.10.1 archived release DOI is
10.5281/zenodo.21270512.
```

## Contribution Basis

Public contributions are handled through the repository's normal open-source
workflow:

- issues for bug reports, enhancement requests, reviews, and questions;
- pull requests for proposed code or documentation changes;
- [`CONTRIBUTING.md`](../CONTRIBUTING.md) for contributor expectations;
- [`SECURITY.md`](../SECURITY.md) for vulnerability reporting.

Submitting an issue, comment, or pull request does not imply endorsement,
sponsorship, employment, scientific validation, or acceptance of a physical
claim.

## Scientific Claim Boundary

QS-DMSS is beta software for reproducible package/evidence workflows. It is
not peer-reviewed scientific validation.

The current public package baseline is `qs-dmss==0.10.1`. Fractal/Quadrant
SSFM scientific feedback is routed through issue #105 before any GPU expansion
or decision-metric UI exposure for spectral diagnostics.

## Names, Marks, And Endorsement

The Apache-2.0 license permits use of the software under its terms. It does not
grant permission to use project or company names, logos, or marks to imply
endorsement, sponsorship, certification, or official validation.

Ordinary nominative references are fine, for example:

```text
Built with QS-DMSS
Uses QS-DMSS v0.10.1
Reproduced with qs-dmss==0.10.1
```

Avoid wording that suggests AI-Bio Synergy Holdings LLC has reviewed,
approved, sponsored, certified, or scientifically validated downstream work
unless there is a separate written agreement or public statement.
